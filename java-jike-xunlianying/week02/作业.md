1.使用 GCLogAnalysis.java 自己演练一遍串行 / 并行 /CMS/G1 的案例。并分析。
## 使用代码
```
	
	import java.util.Random;
	import java.util.concurrent.TimeUnit;
	import java.util.concurrent.atomic.LongAdder;
	/*
	演示GC日志生成与解读
	*/
		public class GCLogAnalysis {
	    private static Random random = new Random();
	    public static void main(String[] args) {
	        // 当前毫秒时间戳
	        long startMillis = System.currentTimeMillis();
	        // 持续运行毫秒数; 可根据需要进行修改
	        long timeoutMillis = TimeUnit.SECONDS.toMillis(1);
	        // 结束时间戳
	        long endMillis = startMillis + timeoutMillis;
	        LongAdder counter = new LongAdder();
	        System.out.println("正在执行...");
	        // 缓存一部分对象; 进入老年代
	        int cacheSize = 2000;
	        Object[] cachedGarbage = new Object[cacheSize];
	        // 在此时间范围内,持续循环
	        while (System.currentTimeMillis() < endMillis) {
	            // 生成垃圾对象
	            Object garbage = generateGarbage(100*1024);
	            counter.increment();
	            int randomIndex = random.nextInt(2 * cacheSize);
	            if (randomIndex < cacheSize) {
	                cachedGarbage[randomIndex] = garbage;
	            }
	        }
	        System.out.println("执行结束!共生成对象次数:" + counter.longValue());
	    }
	
	    // 生成对象
	    private static Object generateGarbage(int max) {
	        int randomSize = random.nextInt(max);
	        int type = randomSize % 4;
	        Object result = null;
	        switch (type) {
	            case 0:
	                result = new int[randomSize];
	                break;
	            case 1:
	                result = new byte[randomSize];
	                break;
	            case 2:
	                result = new double[randomSize];
	                break;
	            default:
	                StringBuilder builder = new StringBuilder();
	                String randomString = "randomString-Anything";
	                while (builder.length() < randomSize) {
	                    builder.append(randomString);
	                    builder.append(max);
	                    builder.append(randomSize);
	                }
	                result = builder.toString();
	                break;
	        }
	        return result;
	    }
	}


```

## 结果分析


内存为256时，串行GC，并行GC，G1 GC频繁发生full gc，最终都会oom，CMS虽然没oom，但也频繁full GC，堆内存也接近用完。

内存较少的范围内，内存越大，发生的fc次数越少，产生的对象越多。

但内存足够大了以后，内存的增加并不会产生更多的对象，但gc的次数也会减少。

内存较小时，串行gc比并行gc发生的gc次数更少，产生的对象更多。此时G1 > CMS > 串行GC > 并行GC

内存较大时，并行GC发挥出很大优势，gc次数很少，并且产生很多的对象。

内存无论大小，G1 GC都产生很多的对象，但GC次数都较多，但基本是young GC，很少或没有full gc，这也是G1 GC虽然gc次数很多，但产生更多的对象的原因。

写一段代码，使用 HttpClient 或 OkHttp 访问 http://localhost:8801 ，代码提交到 Github。

代码
```

	package jike.java.week02;
	
	import okhttp3.OkHttpClient;
	import okhttp3.Request;
	import okhttp3.Response;
	
	public class OkHttpExample {
	    public static void main(String[] args) throws Exception {
	        OkHttpClient client = new OkHttpClient();
	        try {
	            String url = "http://127.0.0.1:8088//api/hello";
	            Request request = new Request.Builder().url(url).build();
	            Response response = client.newCall(request).execute();
	            System.out.println("receive response:");
	            System.out.println(response.body().string());
	        } finally {
	            client.clone();
	        }
	    }
	}
```

```

	hello world
```
